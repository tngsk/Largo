# Lisp / Scheme (Steel) 統合ドキュメント

本システムは、高信頼性・低レイテンシのエッジ音響処理（Rust/C++）と、動的で柔軟な制御層（Lisp/Scheme）を組み合わせたハイブリッド・アーキテクチャの音響エンジンです。このドキュメントでは、本システムにおけるLisp（Steelエンジン）の役割、提供されているAPI、そしてLispを用いることで実現可能な機能について解説します。

## 1. システムにおけるLispの役割と構造

本システムにおいて、Lisp（Scheme）は**「制御層」**および**「動的スクリプティング」**を担当します。
オーディオスレッド（Rust/C++）はリアルタイム性を保証するため、メモリ確保やブロッキング処理が厳密に制限されています。そこで、システム外からの入力（OSCメッセージなど）をパースし、音響パラメータにマッピングする複雑なロジックをメインスレッドのLisp環境で実行し、その結果だけをロックフリーのリングバッファ経由でオーディオスレッドに送る設計となっています。

### データフロー
1. **OSCメッセージ受信**: `127.0.0.1:57120` 経由で他システムからのOSCを受信（Rust層）。
2. **Lispイベント発火**: Rust層が `on-osc-event` を呼び出し、Lisp空間へイベントをディスパッチ。
3. **Lispフック（コールバック）実行**: ユーザーが登録したコールバック関数が、受信したパス（アドレス）と引数を評価。
4. **パラメータ更新指示**: Lisp側から `sound:set-param` を呼び出し、リングバッファ経由でノンブロッキングにパラメータ更新コマンド（`AudioCommand`）を送信。
5. **オーディオ処理**: オーディオスレッドがコマンドを受け取り、RNBOのパラメータを安全に更新して音響を出力。

## 2. Lisp API リファレンス

システムが提供するコアAPIは以下の通りです。

### `(register-osc-hook! callback)`
OSCメッセージを受信した際に実行されるコールバック関数を登録します。
* **引数**: `callback` - `(lambda (path args) ...)` のシグネチャを持つ関数。`path`はOSCアドレスの文字列、`args`は引数のリスト（文字列化されています）。
* **使用例**:
  ```scheme
  (register-osc-hook!
    (lambda (path args)
      (if (equal? path "/sensor/val")
          (display "Sensor input received"))))
  ```

### `(sound:set-param param-name value)`
オーディオスレッド（RNBOモジュール）の指定したパラメータを更新します。
* **引数**:
  * `param-name` (文字列): RNBOモジュール内で定義されたパラメータ名（例: `"frequency"`, `"amplitude"`）。
  * `value` (数値): パラメータに設定する値。
* **特徴**: オーディオスレッドに影響を与えないよう、ロックフリーなリングバッファにコマンドをプッシュします。
* **使用例**:
  ```scheme
  (sound:set-param "frequency" 440.0)
  ```

## 3. スクリプトの配置と規約

* **ファイルの配置**: ユーザー定義のLisp/Schemeスクリプトは `core/scripts/plugins/` ディレクトリ内に `.scm` 拡張子で配置します。システム起動時に自動的に検出・評価されます。
* **名前空間の衝突回避**: 複数のプラグインが共存できるよう、定義する関数名にはファイル名やプラグイン名に基づいたプレフィックス（コロン区切り）を付けることが推奨されます（例: `(define (sensor-synth:map ...))`）。

## 4. Lispで実現できること（可能性と応用例）

Lisp層は汎用的なプログラミング言語（Scheme）であるため、単なる1対1のパラメータマッピングを超えた、高度で柔軟な音響制御が可能です。

### 4-1. 動的で非線形なパラメータマッピング
センサーから送られてくる生データをそのまま音響パラメータにするのではなく、Lisp内でスケーリング、クランプ、非線形カーブ（指数・対数）の適用などを行ってから `sound:set-param` を呼び出すことができます。
```scheme
;; センサー値 (0.0 - 1.0) を周波数 (100Hz - 2100Hz) にマッピング
(define (my-plugin:map-freq val)
  (+ 100.0 (* val 2000.0)))
```

### 4-2. 条件付きパラメータルーティング
受信したOSCのパスや値、あるいは特定の状態に応じて、操作するパラメータを動的に切り替えることができます。「値が一定閾値を超えた場合のみエフェクトを強める」といった振る舞いが簡単に記述できます。

### 4-3. 複数パラメータへの同時展開（マクロ機能）
1つのOSCメッセージを受け取って、複数のRNBOパラメータを同時に操作する「マクロ」のような振る舞いを実装できます。これにより、外部システムからは1つの抽象的な指示（例: `/mood/tension 0.8`）を送り、Lisp側でそれを「ピッチ上昇」「ノイズ追加」「リバーブ増大」などに分解してオーディオエンジンに指示を出すことが可能です。

### 4-4. 状態の保持とアルゴリズミックな処理
Lisp内でグローバル変数（状態）を保持することで、過去の入力履歴に基づいた処理（移動平均の算出によるノイズ除去、トグルの実装、ステップシーケンサー的な状態遷移）など、より高度なインタラクションルールを現場の要件に合わせて記述できます。

### 4-5. 安全なライブコーディング的アプローチへの拡張性
本アーキテクチャではオーディオスレッドとロジック（Lisp）が分離されているため、将来的にLispスクリプトを動的に再読み込み（ホットリロード）する仕組みを導入すれば、音を止めずにコンポジションルールを更新するライブコーディング的な運用も容易に実現可能です。
